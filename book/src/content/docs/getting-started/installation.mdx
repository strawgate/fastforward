import { Tabs, TabItem } from '@astrojs/starlight/components';

:::note
The product is **logfwd**. The GitHub repository is named `memagent` for historical reasons. Docker images, URLs, and `git clone` commands reference `memagent`; the binary and all user-facing output say `logfwd`.
:::

<Tabs>
  <TabItem label="From Source" icon="seti:rust">
    Requires the Rust stable toolchain (1.85+).

    ```bash
    git clone https://github.com/strawgate/memagent.git
    cd memagent
    cargo build --release -p logfwd
    ```

    The binary is at `./target/release/logfwd`. Copy it wherever you need:

    ```bash
    sudo cp target/release/logfwd /usr/local/bin/
    ```
  </TabItem>
  <TabItem label="Docker" icon="docker">
    ```bash
    docker run --rm \
      -v /var/log:/var/log:ro \
      -v $(pwd)/config.yaml:/etc/logfwd/config.yaml:ro \
      ghcr.io/strawgate/memagent:latest run --config /etc/logfwd/config.yaml
    ```

    Images are published to `ghcr.io/strawgate/memagent` for `linux/amd64` and `linux/arm64`. See [Docker deployment](/memagent/deployment/docker/) for compose files and volume configuration.
  </TabItem>
  <TabItem label="Kubernetes" icon="cloud">
    A ready-to-apply DaemonSet manifest is included in the repo:

    ```bash
    kubectl apply -f deploy/daemonset.yml
    ```

    See the [Kubernetes deployment guide](/memagent/deployment/kubernetes/) for resource sizing and collector integration.
  </TabItem>
  <TabItem label="Binary" icon="linux">
    :::caution
    Pre-built release binaries are not yet published. Use **From Source** or **Docker** for now. When releases are available, binaries will be downloadable from [GitHub Releases](https://github.com/strawgate/memagent/releases).
    :::

    Planned artifacts:

    | Platform | Artifact |
    |----------|----------|
    | Linux x86_64 | `logfwd-linux-amd64` |
    | Linux ARM64 | `logfwd-linux-arm64` |
    | macOS x86_64 (Intel) | `logfwd-darwin-amd64` |
    | macOS ARM64 (Apple Silicon) | `logfwd-darwin-arm64` |
  </TabItem>
</Tabs>

## Verify installation

```bash
logfwd --version
logfwd --help
```
